# 小红书内容拉取 — 参考说明

## 官方与合规渠道

- 查阅 **小红书开放平台** 最新文档与准入条件（搜索「小红书开放平台」获取当前域名与产品说明）。  
- 企业/合作场景通常需单独申请接口权限；个人开发者以平台公示为准。

## 环境变量（示例占位）

在实现真实调用前，由用户在环境中配置（名称可按你方开放平台文档调整）：

| 变量 | 说明 |
|------|------|
| `XHS_APP_ID` | 应用 ID |
| `XHS_APP_SECRET` | 应用密钥（勿提交到 Git） |
| `XHS_API_BASE` | API 网关基地址（以官方文档为准） |

## 接口形态

本仓库**不包含**真实签名算法或私有端点实现。`xiaohongshu-content-fetch/scripts/fetch_stub.py` 仅演示：

1. 读取 `prompt_to_params.py` 产出的 JSON  
2. 在已设置 `XHS_API_BASE` 等变量时，可扩展为真实 `requests` 调用  

具体路径、鉴权方式、限流以官方文档为准。

## 与 Cursor Skill 的关系

- Skill 指导 Agent：**如何理解提示词、如何组织输出、合规边界**。  
- 代码目录 `xiaohongshu-content-fetch/`：本地可运行的解析与占位拉取逻辑，便于你接入真实 API。
